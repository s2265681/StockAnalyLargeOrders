# 自动注册实现原理

本文档详细解析脚本如何模拟浏览器完成网站注册，适合学习 HTTP 协议、Web 安全机制和 Python 爬虫技术。

## 总体思路

用户在浏览器中注册时，浏览器做了以下事情：

```
打开注册页面 → 获取 Cookie → 填写表单 → 点击发送验证码 → 输入验证码 → 点击注册
```

脚本做的完全一样，只是把"浏览器操作"替换成了"HTTP 请求"：

```
requests.get(注册页) → 提取令牌 → requests.post(发短信) → e码收短信 → requests.post(注册)
```

核心就一句话：**用代码重放浏览器的 HTTP 请求序列**。

---

## 第1步：访问注册页面 — 获取 Cookie 和动态令牌

```python
resp = self.session.get("https://dapaodan.com/register/")
```

### 发生了什么

1. **服务器设置 Cookie**：返回 `wp_xh_session_xxx` Cookie，标识这是一个新访客。后续所有请求都必须带上这个 Cookie，否则服务器不认识你。

2. **页面 HTML 中嵌入了安全令牌**：这是 WordPress 的 CSRF（跨站请求伪造）防护机制。页面的 JavaScript 代码中包含了几个一次性令牌：

```javascript
// 发送验证码时需要的令牌（在 JS 对象中）
{
  "xh_social_wechat_social_add_ons_social_mobile": "4a095afdda",  // nonce
  "notice_str": "7451677357",
  "hash": "6ef845e6c1f1d3198a69ccaae1fbce14"
}

// 提交注册时需要的令牌（在 URL 中）
"...?action=xh_social_add_ons_login&tab=register&xh_social_add_ons_login=261c223708&notice_str=9751737557&hash=0a0f36254950ffbf02f8f35f756d5cc1"
```

### 为什么需要这些令牌

这是 **CSRF 防护**：服务器生成的随机值，嵌入页面中。提交表单时必须带上这些值，服务器验证后才处理请求。这确保请求来自"真正访问过注册页面的人"，而不是来自第三方网站的恶意请求。

### 我们怎么做

用正则表达式从 HTML 中提取这些令牌：

```python
# 从 JS 对象中提取
re.search(r'"xh_social_wechat_social_add_ons_social_mobile"\s*:\s*"([a-f0-9]+)"', html)

# 从 URL 字符串中提取
re.search(r'action=xh_social_add_ons_login&tab=register&xh_social_add_ons_login=([a-f0-9]+)&notice_str=(\d+)&hash=([a-f0-9]+)', html)
```

### 关键技术：requests.Session()

```python
self.session = requests.Session()
```

`Session` 对象会**自动管理 Cookie**。第一次 GET 请求拿到的 Cookie，后续 POST 请求会自动带上，无需手动处理。这和浏览器的行为一致。

## 第2步：获取手机号 — e码平台 API

```python
self.ema.get_phone(card_type='实卡', check_availability=False)
```

### e码平台是什么

e码是一个**虚拟手机号平台**，提供可编程的 API，核心功能：

| API | 作用 | 示例 |
|-----|------|------|
| `code=login` | 登录获取 token | 返回 `8a41fe4740...` |
| `code=getPhone` | 获取一个手机号 | 返回 `18589564519` |
| `code=getMsg` | 查询该号码收到的短信 | 返回短信内容 |
| `code=release` | 释放手机号（归还） | 用完后调用 |

### API 调用示例

```
GET http://api.eomsg.com/zc/data.php?code=getPhone&token=xxx&cardType=实卡

返回：18589564519
```

### 为什么用实卡

e码有两种号码：`实卡`（真实 SIM 卡）和 `虚卡`（虚拟号码）。一些网站会检测虚拟号码段并拒绝发送短信，实卡的成功率更高。

## 第3步：发送短信验证码 — 模拟点击"发送验证码"按钮

```python
self.session.post("https://dapaodan.com/wp-admin/admin-ajax.php", data=data)
```

### WordPress 的 AJAX 机制

WordPress 所有的 AJAX 请求都走 **`/wp-admin/admin-ajax.php`** 这个统一入口，通过 `action` 参数区分不同的操作：

- `action=xh_social_wechat_social_add_ons_social_mobile` → 发送短信验证码
- `action=xh_social_add_ons_login&tab=register` → 提交注册

### 请求内容

```python
data = {
    'action': 'xh_social_wechat_social_add_ons_social_mobile',
    'tab': 'mobile_login_vcode',
    'xh_social_wechat_social_add_ons_social_mobile': '4a095afdda',  # 第1步提取的 nonce
    'hash': '6ef845e6c1f1d3198a69ccaae1fbce14',                     # 第1步提取的 hash
    'notice_str': '7451677357',                                      # 第1步提取的 notice_str
    'unique': '1',
    'mobile': '18589564519',     # 第2步获取的手机号
    'register_user_login': 'yanglin',
    'register_user_email': 'yanglin@sina.com',
    'register_password': 'Wei1991b',
}
```

### Headers 的切换

访问注册页面时，浏览器发送的是页面请求（`Accept: text/html`）。点击按钮后，变成 AJAX 请求，Headers 需要相应切换：

```python
self.session.headers.update({
    'Accept': 'application/json, text/javascript, */*; q=0.01',  # 告诉服务器期望 JSON
    'Content-Type': 'application/x-www-form-urlencoded; charset=UTF-8',  # 表单编码
    'X-Requested-With': 'XMLHttpRequest',  # 标识这是 AJAX 请求
    'Referer': 'https://dapaodan.com/register/',  # 来源页面
})
```

`X-Requested-With: XMLHttpRequest` 是关键——WordPress 用它判断请求是否来自前端 AJAX 调用。

### 服务器的响应

```json
{"errcode": 0, "errmsg": "", "data": null}
```

`errcode: 0` 表示成功，服务器已向手机号发送短信。

## 第4步：接收验证码 — 轮询 e码 API

```python
params = {
    'code': 'getMsg',
    'token': self.ema.token,
    'phone': '18589564519',
    'keyWord': '大炮弹',      # 短信签名中【】里的内容
}
resp = self.ema.session.get(self.ema.api_url, params=params)
```

### 轮询机制

短信不是实时到达的，需要**定时查询**：

```
每3秒查询一次 → 没收到 → 继续等 → 收到 → 提取验证码
```

超时设置为 90 秒，超时未收到则失败。

### keyWord 参数

e码 API 要求指定**短信关键词**来过滤短信。关键词就是短信中 `【】` 里的发送者名称。例如短信内容是：

```
【佛山市大炮弹投资咨询】您的验证码为：787171，请勿泄露于他人！
```

关键词填 `大炮弹` 即可匹配。

### e码返回格式

```
18589564519/0.28/【佛山市大炮弹投资咨询】您的验证码为：787171，请勿泄露于他人！
```

格式是 `手机号/费用/短信内容`，用 `/` 分隔。需要取第三部分再提取验证码。

### 验证码提取

用正则表达式从短信正文中提取 4-6 位数字：

```python
patterns = [
    r'验证码[：:]\s*(\d{4,6})',   # 匹配 "验证码：787171"
    r'码[：:]\s*(\d{4,6})',       # 匹配 "码：787171"
    r'验证码是(\d{4,6})',         # 匹配 "验证码是787171"
    r'(\d{4,6})',                 # 兜底：任意 4-6 位数字
]
```

## 第5步：提交注册 — 最终请求

```python
# 注册的令牌放在 URL 参数中
params = {
    'action': 'xh_social_add_ons_login',
    'tab': 'register',
    'xh_social_add_ons_login': '261c223708',    # 第1步提取的 nonce
    'notice_str': '9751737557',
    'hash': '0a0f36254950ffbf02f8f35f756d5cc1',
}

# 用户信息和验证码放在 POST body 中
data = {
    'register_user_login': 'yanglin',
    'register_user_email': 'yanglin@sina.com',
    'register_password': 'Wei1991b',
    'mobile': '18589564519',
    'mobile_vcode': '787171',                    # 第4步拿到的验证码
}

resp = self.session.post(AJAX_URL, params=params, data=data)
```

服务器验证所有信息（令牌、用户名唯一性、邮箱格式、短信验证码）后返回：

```json
{"errcode": 0, "errmsg": "", "data": null}
```

注册成功。

## 涉及的核心技术总结

### 1. HTTP 协议基础

| 概念 | 作用 | 对应代码 |
|------|------|---------|
| GET 请求 | 获取页面内容 | `session.get(url)` |
| POST 请求 | 提交表单数据 | `session.post(url, data=...)` |
| Cookie | 维持会话状态 | `Session()` 自动管理 |
| Headers | 模拟浏览器身份 | `session.headers.update(...)` |
| URL 参数 | 附在 URL 后面的键值对 | `params={}` |
| POST body | 请求体中的表单数据 | `data={}` |

### 2. Web 安全机制

| 机制 | 原理 | 我们的应对 |
|------|------|----------|
| CSRF 令牌 | 服务器生成随机值嵌入页面，提交时验证 | 正则从 HTML 提取 |
| Cookie 验证 | 服务器通过 Cookie 识别用户身份 | Session 自动维护 |
| Referer 检查 | 检查请求来源页面 | Headers 中设置 Referer |
| AJAX 标识 | 检查 X-Requested-With 头 | Headers 中设置对应值 |

### 3. 正则表达式

```python
# 从 JS 代码中提取 nonce 值
r'"xh_social_wechat_social_add_ons_social_mobile"\s*:\s*"([a-f0-9]+)"'
#                                                         ^^^^^^^^^^
#                                                         捕获组：匹配十六进制字符串

# 从 URL 中提取多个参数
r'xh_social_add_ons_login=([a-f0-9]+)&notice_str=(\d+)&hash=([a-f0-9]+)'
#                          ^group(1)               ^group(2)   ^group(3)
```

### 4. 第三方 API 集成（e码平台）

```
登录 → 获取 token → 取号 → 发送验证码 → 轮询取码 → 释放号码
```

这是典型的 **token 认证** + **轮询模式**：先登录拿凭证，再用凭证调用其他接口。

## 学习建议

1. **先学 HTTP 基础**：理解 GET/POST、Cookie、Headers 的作用
2. **用浏览器 DevTools 抓包**：F12 → Network 面板，观察浏览器注册时发了哪些请求
3. **对比脚本和抓包结果**：理解脚本中的每个参数对应浏览器的哪个字段
4. **尝试修改目标网站**：换一个 WordPress 站点，调整令牌提取的正则
5. **深入 Web 安全**：了解 CSRF、XSS、CORS 等概念，理解防护机制的设计思路
